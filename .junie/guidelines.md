# Spec-Driven Development Guidelines
This project follows the GitHub Spec Kit workflow. Before implementing code:
1. Refer to `.specify/constitution.md` for project rules.
2. Ensure a feature has a valid `spec.md` and `plan.md`.
3. Only implement code based on the checkboxes in `tasks.md`.

You are authorized to use Spec Kit commands by reading the templates in `.specify/templates/`.
